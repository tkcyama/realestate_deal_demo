# PropConnect — 不動産取引プラットフォーム

## プロジェクト概要

法人向け不動産取引プラットフォーム。不動産ファンド・レンダーを対象に、物件情報管理・オファー取引・AI解析・取引事例DBを一元管理する。

## 技術スタック

- **フロントエンド**: Next.js (TypeScript) + Tailwind CSS + shadcn/ui
- **バックエンド/DB**: Supabase (PostgreSQL + Auth + Storage + RLS)
- **地図**: Google Maps JavaScript API（Geocoding API含む）
- **AI**: Claude AI API（claude-sonnet-4-6）
- **PDF生成**: Puppeteer（HTML → PDF）
- **メール通知**: Resend API
- **ホスティング**: Vercel + Supabase
- **CI/CD**: GitHub Actions + Vercel

## 対象ユーザーとロール

1つの会員アカウントが複数ロールを同時保持可能（案件ごとに切り替わる）。

| ロール | 説明 |
|--------|------|
| 売主 | 物件を登録・公開し、買主にオファーを送る |
| 買主 | 物件を閲覧し、購入オファーを送る |
| レンダー | 取引合意後にローンオファーを受け取る |
| 管理者 | 会員申請の承認・物件公開の最終承認 |

## 実装方針（決定事項）

- **DBスキーマを全フェーズ分まとめて先に設計・Supabaseに入れる**（RLSを後付けすると改修コストが大きいため）
- Phase 1（会員管理）→ Phase 2（地図・物件）の順に進める
- `realestate_deal_demo/index.html` のモックUIを Next.js コンポーネントへ移植しながら実装する

## 重要設計ポイント

### セキュリティ（最優先）
- Supabase RLS でロール・オファーステータスに応じたアクセス制御を厳格に実装
- 物件詳細情報はオファー受理後のみ閲覧可能
- 取引事例はテナント名・個人情報を匿名化して公開

### 地図ピンの色分け
| 種別 | 色 |
|------|-----|
| 保有物件 | 紺（#1F3864） |
| 売却物件 | 赤（#C00000） |
| 取引事例 | 緑（#1F6B3A） |

### AI処理（Phase 4）
- Claude API で PDF/Excel/Word からNOI・NCF・RR等を自動抽出
- 抽出結果は確認画面を経てDB登録（自動確定しない）
- Teaser生成は30秒以内を目標
- 精度目標：主要財務項目（NOI・NCF）で95%以上

## 認証フロー
1. Web申請フォーム送信
2. 確認メール送信
3. 管理者が審査・承認/却下
4. 承認後にパスワード設定メール送信
5. 初回ログイン時にロール選択（複数可）

## 物件管理 仕様詳細（実装確定事項）

### 物件ステータス遷移

```
draft（下書き）
  └─[公開申請]──→ pending_approval（承認待ち）
                      ├─[申請取り消し]──→ draft
                      └─[管理者承認]───→ published（公開中）
                                              └─[非公開に戻す]──→ draft
sold（売却完了）← Phase 3 以降で遷移
```

- `published` → `draft` への非公開化は売主本人が実行可能
- `sold` は売主・管理者では変更不可（Phase 3 の取引完了フローで自動遷移）
- 編集フォームは `draft` / `pending_approval` のみアクセス可能

### 価格の入力・表示単位

| 項目 | 入力単位 | DB保存単位 | 表示単位 |
|------|---------|-----------|---------|
| 売却希望価格 | 億円（例: 50） | 円（例: 5,000,000,000） | 億円（例: 50億円） |
| NOI 年間純収益 | 円 | 円 | 円（formatPrice） |
| NCF 純キャッシュフロー | 円 | 円 | 円（formatPrice） |
| 月額賃料収入 | 円 | 円 | 円（formatPrice） |

- NOI利回り・NCF利回りはすべて円単位で計算（`noi / price`）
- `formatPropertyPrice()`: 億円単位で表示（100億以上は整数、1億以上は小数1桁、1億未満は小数2桁）

### 住所正規化ルール（`src/lib/normalize.ts`）

物件登録・編集の保存時に全住所フィールド（都道府県・市区町村・町域・番地以降）へ適用。

| 変換対象 | 変換例 |
|---------|-------|
| 全角数字 → 半角 | `３９` → `39` |
| 全角英字 → 半角 | `Ａ棟` → `A棟` |
| 全角ハイフン/長音 → 半角ハイフン | `－` / `ー` → `-` |
| 漢数字（単体） → 半角 | `三丁目` → `3丁目` |
| 漢数字（複合） → 半角 | `三十九番` → `39番`、`百二十三` → `123` |
| 番(地)?数字(号) → ハイフン | `39番7号` / `39番地7号` → `39-7` |
| 末尾の番地を除去 | `39番地` → `39` |
| 連続ハイフンを統一 | `39--7` → `39-7` |

**誤変換しないケース**: `番町`（地名）・`号館`・`号室` は変換対象外

### 物件一覧フィルター仕様

| フィルター | 実装方式 | 対象 |
|-----------|---------|------|
| キーワード | `name` / `address` 部分一致（ilike） | 保有物件・公開中物件 |
| 用途 | 完全一致 | 同上 |
| 都道府県 | 部分一致（ilike） | 同上 |
| 価格帯（億円） | 下限: `≥ price_min × 1億`、上限: `≤ price_max × 1億` | 同上 |

- フィルター状態は URL パラメータで管理（ブックマーク・共有対応）
- フィルターなし時はクリアボタン非表示
