## 実施作業概要

### Phase 1：基盤構築・会員管理

#### 1-1. プロジェクトセットアップ
- `create-next-app` による Next.js プロジェクト初期化
- Tailwind CSS・shadcn/ui・各種依存パッケージのインストール
- `.env.local` への Supabase / Google Maps API キー設定

#### 1-2. DBスキーマ設計・デプロイ（全フェーズ分）
- **設計方針**: RLS後付け改修コストを避けるため、全フェーズ分のスキーマを一括設計・先行デプロイ
- `supabase/migrations/0001_initial_schema.sql`（911行）として実装
  - **テーブル数**: 17（profiles, properties, office_details, residential_details, commercial_details, logistics_details, hotel_details, sale_offers, purchase_offers, loan_offers, documents, ai_extraction_jobs, property_images, transactions, transaction_details, notifications, audit_logs）
  - **ENUM数**: 10
  - **インデックス数**: 43
  - **RLSポリシー数**: 54
  - **ヘルパー関数**: 5（`is_approved_member()`, `is_admin_user()`, `has_property_detail_access()` 等）
  - **トリガー数**: 13（新規ユーザー登録時の profiles 自動生成 等）
  - **Storageバケット**: 3（property-images, documents, teasers）

#### 1-3. 認証・会員管理フロー実装
- Supabase Auth (`@supabase/ssr`) を用いたログイン・新規登録ページ
- `src/middleware.ts` によるルート保護（未認証→ /auth/login、管理者専用ルートのガード）
- `handle_new_user()` トリガーによる profiles 自動生成
- 管理者承認フロー（`profiles.status` = pending → approved）
- 審査中・停止ユーザー向けブロック画面（dashboard layout）

#### 1-4. 管理者：会員管理画面
- `/admin/members` — 会員一覧・承認/却下/停止/再有効化ボタン
- `member-actions.tsx` — Supabase update + `router.refresh()` によるリアルタイム反映

### Phase 2（一部）：物件管理・地図表示

#### 2-1. Google Maps 地図表示
- `@vis.gl/react-google-maps` を使用
- AdvancedMarker + Pin によるピン色分け
  - 紺（#1F3864）: 自分の保有物件
  - 赤（#C00000）: 公開中物件
  - 緑（#1F6B3A）: 取引事例
- InfoWindow ポップアップ（物件名・住所・用途・価格・NOI利回り）
- 凡例コンポーネント（地図左下）
- Google Geocoding API による住所→緯度経度変換（`src/lib/geocoding.ts`）

#### 2-2. 物件登録フォーム（`/properties/new`）
- react-hook-form + zod によるバリデーション
- **入力セクション構成**:
  - 基本情報（物件名・所在地・用途・建築年・構造・地上/地下階数）
  - 所在地（都道府県・市区町村・町域・番地以降に分割保持）
  - 交通（路線・最寄り駅・徒歩分数）
  - 面積情報（延床・専有・土地）
  - 財務情報（価格・NOI・NCF・月額賃料収入）
  - テナント情報（テナント数・稼働率）
- 自動計算: NOI利回り・NCF利回り・専有坪単価・専有面積（㎡換算）
- 下書き保存 / 公開申請 の2段階ボタン
- バリデーションエラー時に上部バナーで具体的な項目名を表示・自動スクロール

#### 2-3. 物件詳細ページ（`/properties/[id]`）
- 物件情報の表示
- 売主本人向け: 「公開申請」「申請取り消し」ステータス操作ボタン
- エラー表示・更新失敗時のメッセージ

#### 2-4. 管理者：物件承認管理（`/admin/properties`）
- 承認待ち物件一覧（バッジで件数表示）
- 「承認・公開」「差し戻し」ボタン

## 解決した主な技術的問題

| 問題 | 原因 | 解決策 |
|------|------|--------|
| `useSearchParams()` ビルドエラー | Next.js 16 で Suspense 必須 | ログイン・プロフィールページに `<Suspense>` ラッパー追加 |
| zod + react-hook-form 型エラー | `z.coerce.number()` が `unknown` を返す | 全数値フィールドを `z.string().refine()` に統一 |
| Supabase `.select()` API エラー | 第2引数に `count` オプション不可 | `.select('id')` のみに変更 |
| 公開申請ボタン無反応 | バリデーションエラーがコンソールのみ出力 | 上部バナーに具体的なエラー項目を一覧表示・自動スクロール |
| 地下階数バリデーションエラー | フォームにUI入力欄なく `undefined` が渡る | 地下階数の入力欄を追加（地上/構造/建築年と同列に配置） |
| 管理者の初回ログイン | 審査中のため管理画面にアクセス不可 | Supabase SQL Editor で直接 `is_admin = true` に更新 |
